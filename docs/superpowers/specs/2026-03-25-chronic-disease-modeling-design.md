# 慢性病风险因素分析与预测模型研究主线设计

## 1. 背景

本设计文档对应毕业设计《慢性病风险因素分析与预测模型研究》的第一阶段实现范围。根据开题报告，整体课题包含两类交付：

1. 研究建模主线
2. 可演示的网页界面

当前优先启动研究建模主线，不在本阶段实现网页。网页将在后续阶段直接消费本阶段产出的标准化模型文件、指标结果和解释性结果。

## 2. 设计目标

第一阶段的目标是构建一个可复现、可扩展、可验证的科研建模仓库，完成以下闭环：

1. 获取并整理 NHANES 数据
2. 完成数据清洗与变量对齐
3. 计算 GLM7 及相关派生指标
4. 构建多病种风险预测模型
5. 进行单队列时间验证
6. 输出评估指标、图表和 SHAP 解释结果
7. 为后续 CHARLS 外部验证和网页演示预留稳定接口

## 3. 范围定义

### 3.1 本阶段包含

1. NHANES 数据下载与读取
2. 面向建模的数据清洗流程
3. GLM7、AIP、TyG、TyG-BMI 等指标计算
4. 心血管疾病、糖尿病、肝病、癌症、共病风险五类预测任务
5. 逻辑回归、XGBoost、随机森林、SVM 模型训练与比较
6. 基于 NHANES 周期的时间切分验证
7. SHAP 解释性分析
8. 面向网页消费的轻量化结果导出

### 3.2 本阶段不包含

1. CHARLS 正式建模与正式外部验证
2. 网页前后端实现
3. 论文终稿撰写
4. 与医院或真实业务系统对接

### 3.3 预留能力

1. CHARLS 数据源接入层
2. CHARLS 字段映射与验证入口
3. 网页演示所需的标准产物格式

## 4. 关键设计决策

### 4.1 先做 NHANES 单队列完整闭环

理由：

1. 当前没有现成数据，先做 NHANES 能最快形成可运行成果。
2. 指导教师意见明确建议先做单队列时间验证，避免直接跨队列验证引入偏倚。
3. 后续拿到 CHARLS 后，只需补接入与适配，不必推翻整体结构。

### 4.2 研究主线与网页演示解耦

研究代码仓库负责产生标准化结果文件，不在网页中重复实现核心算法。这样可以避免：

1. 逻辑分叉
2. 实验结果与网页结果不一致
3. 后续论文结果难以复现

### 4.3 Notebook 不承载正式主逻辑

正式逻辑放在 `src/` 与 `scripts/` 中，Notebook 只用于探索分析和补充展示。这样能保证流程可测试、可自动化、可复跑。

## 5. 总体架构

研究主线按六个模块组织：

1. `data_sources`
   - 下载、登记、读取 NHANES 原始数据
   - 为 CHARLS 预留接入接口

2. `preprocessing`
   - 变量筛选
   - 缺失值处理
   - 异常值处理
   - 纳入排除标准执行
   - 标签定义

3. `features`
   - GLM7 七项指标计算
   - AIP、TyG、TyG-BMI 等派生特征构造
   - 特征校验

4. `modeling`
   - 基线模型与机器学习模型训练
   - 交叉验证与调参
   - 时间切分验证

5. `evaluation`
   - AUC、Accuracy、Precision、Recall、F1 等指标计算
   - ROC 曲线、混淆矩阵、阈值分析
   - 结果汇总导出

6. `interpretability`
   - SHAP 解释
   - 特征重要性可视化
   - 单样本解释结果导出

## 6. 数据流设计

数据流按“原始数据只读、中间结果可检查、最终产物可追溯”的原则组织。

### 6.1 数据分层

1. `data/raw`
   - 原始下载数据
   - 不做人工修改

2. `data/interim`
   - 原始表合并后的中间结果
   - 基础清洗结果
   - 字段映射结果

3. `data/processed`
   - 可直接建模的数据集
   - 包含标签、GLM7、派生特征、切分结果

4. `models`
   - 训练得到的模型文件
   - 参数配置与元信息

5. `reports`
   - 论文和汇报所需图表与表格

6. `artifacts`
   - 网页消费的轻量化结果文件

## 7. 建模任务设计

### 7.1 预测任务

第一阶段按五个单独任务组织：

1. 心血管疾病风险预测
2. 糖尿病风险预测
3. 肝病风险预测
4. 癌症风险预测
5. 共病风险预测

每个任务独立训练、独立评估、独立导出结果。这样更便于：

1. 控制任务边界
2. 撰写论文结果
3. 后续网页逐项展示风险

### 7.2 模型策略

模型分两层：

1. 基线模型
   - 逻辑回归
   - 作为基础对照与解释性参考

2. 主力模型
   - XGBoost
   - RandomForest
   - SVM

调参在训练集内部完成，不使用测试集调参。

### 7.3 验证策略

第一阶段采用以下策略：

1. 训练集内部交叉验证
   - 用于模型选择和超参数调优

2. NHANES 时间切分验证
   - 使用较早周期训练
   - 使用较晚周期测试
   - 作为正式报告性能

3. CHARLS 外部验证预留
   - 暂不执行正式实验
   - 预留字段匹配与评估入口

### 7.4 数据泄漏控制

以下处理必须只在训练集上拟合：

1. 缺失值插补
2. 标准化或缩放
3. 编码转换
4. 特征选择
5. 任何依赖数据分布的变换

测试集和后续外部验证集只允许执行 `transform` 与 `predict`。

## 8. 结果产物设计

每个预测任务都应产出以下结果：

1. 模型文件
2. 模型配置与参数信息
3. 指标汇总表
4. ROC 图
5. 混淆矩阵图
6. 阈值分析结果
7. SHAP 图
8. 给网页使用的 JSON 或其他轻量化描述文件

网页阶段应直接读取这些产物，而不是重新训练模型。

## 9. 异常处理设计

### 9.1 数据下载失败

要求：

1. 记录失败文件名、来源、时间
2. 支持重试
3. 不允许静默失败

### 9.2 字段缺失或周期差异

要求：

1. 读取层执行字段存在性检查
2. 输出缺失字段清单
3. 阻止错误字段进入后续建模

### 9.3 GLM7 计算异常

典型问题：

1. 除零
2. 对数非法值
3. BMI 缺失
4. 胰岛素缺失

要求：

1. 统一处理为缺失值或显式异常
2. 记录异常样本数量
3. 不允许直接输出错误结果

### 9.4 模型训练失败

要求：

1. 单个模型失败时记录原因
2. 不中断整批任务
3. 继续执行其他模型训练与评估

### 9.5 结果文件写出失败

要求：

1. 所有产物写出后进行存在性校验
2. 如果导出失败，流程显式报错

## 10. 复现性要求

1. 所有路径、周期范围、随机种子、目标任务放入 `configs/`
2. 所有正式流程由命令行脚本触发
3. 每一步结果带元信息
4. 模型与评估结果可一一追溯
5. 正式结果不依赖 Notebook 手工操作

## 11. 测试方案

### 11.1 单元测试

覆盖以下内容：

1. AIP 公式计算
2. TyG 公式计算
3. TyG-BMI 公式计算
4. GLM7 组合逻辑

### 11.2 数据契约测试

覆盖以下内容：

1. NHANES 输入字段是否齐全
2. 字段类型是否符合要求
3. 标签定义输入是否满足条件

### 11.3 流水线最小闭环测试

使用小样本数据跑通：

1. 读取
2. 清洗
3. 特征构造
4. 模型训练
5. 结果评估
6. 产物导出

### 11.4 结果校验测试

覆盖以下内容：

1. 模型输出概率是否在合法范围
2. 指标表字段是否完整
3. 图表文件是否生成
4. 网页消费文件是否生成

### 11.5 回归测试

后续补充 CHARLS 或网页阶段时，保证 NHANES 主链路不被破坏。

## 12. 非目标与风险

### 12.1 非目标

1. 第一阶段不追求直接完成跨队列正式结论
2. 第一阶段不实现完整网页系统
3. 第一阶段不引入复杂深度学习模型

### 12.2 当前风险

1. NHANES 与 CHARLS 字段口径可能不完全一致
2. 部分指标在不同周期中可能存在缺失或命名变化
3. 共病标签定义需要在实现前进一步细化
4. 参考文献中的 GLM7 文献可能需要再次核对公式与阈值来源

## 13. 下一阶段实施边界

本设计通过后，实施计划应优先完成以下顺序：

1. 项目脚手架与配置结构
2. NHANES 下载与读取
3. 指标与标签计算
4. 最小建模闭环
5. 多模型比较与结果导出
6. SHAP 解释
7. 网页消费产物导出

CHARLS 接入和网页演示均属于后续阶段，不纳入本阶段首轮实现。

## 14. 当前结论

本课题第一阶段的正确启动方式不是直接做网页，也不是立即强行做双队列研究，而是：

1. 先建立 NHANES 单队列、可复现、可验证的研究建模主线
2. 将 CHARLS 作为后续外部验证阶段接入
3. 将网页作为消费研究结果的展示层单独设计

该方案与当前数据准备状态一致，也更符合开题报告与指导教师意见。
